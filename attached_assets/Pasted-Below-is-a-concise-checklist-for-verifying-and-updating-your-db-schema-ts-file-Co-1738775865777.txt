Below is a concise checklist for verifying and updating your `db/schema.ts` file:

---

### **Column Definitions & Data Types**
- [ ] **Products Table:**
  - Verify required columns (e.g., `id`, `userId`, `name`, `price`, `quantity`, etc.) are present with correct types.
  - Confirm that text fields like `name`, `description`, `sku`, `condition`, etc., are defined as `text`.
  - Ensure numeric fields (`price`, `weight`, etc.) are defined using `decimal` with appropriate precision (10) and scale (2).
  - Check timestamp fields (`createdAt`, `updatedAt`, `listedAt`, `soldAt`, `ebayLastSync`) are defined correctly, with defaults set where needed.
  - **Sold Column:**  
    - Confirm the `sold` column is present, defined as a `boolean` with a default value of `false` and marked as not null.
  - **eBay Fields:**  
    - Verify the inclusion of:
      - `ebayListingId` (text)
      - `ebayListingStatus` (text)
      - `ebayListingUrl` (text)
      - `ebayListingData` (jsonb)
      - `ebayLastSync` (timestamp)
      - `ebayCategoryId` (text)

- [ ] **Users, Watchlist, Orders, OrderItems Tables:**
  - Confirm that all required columns exist and have appropriate types.
  - Verify unique constraints where needed (e.g., `username` in `users` and `sku` in `products`).

---

### **Foreign Key Relationships**
- [ ] **Products Table:**
  - Ensure `userId` references `users.id` with `onDelete: 'cascade'`.
- [ ] **Watchlist Table:**
  - Verify `userId` references `users.id` with `onDelete: 'cascade'`.
  - Confirm `productId` references `products.id` with `onDelete: 'cascade'`.
- [ ] **Orders Table:**
  - Ensure `userId` references `users.id` with `onDelete: 'cascade'`.
- [ ] **OrderItems Table:**
  - Check `orderId` references `orders.id` with `onDelete: 'cascade'`.
  - Confirm `productId` references `products.id` with `onDelete: 'set null'` (to preserve order history after product deletion).

---

### **Relations & Schema Exports**
- [ ] **Relations:**
  - Verify that relationships are correctly defined using `relations` (e.g., `productsRelations`, `watchlistRelations`, `ordersRelations`, and `orderItemsRelations`).
- [ ] **Zod Schemas:**
  - Confirm that insert and select schemas are generated using `createInsertSchema` and `createSelectSchema` for all tables.
- [ ] **Type Exports:**
  - Ensure that exported types (e.g., `InsertUser`, `SelectProduct`, etc.) correctly reflect the schema definitions.
